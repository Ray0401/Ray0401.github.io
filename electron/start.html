现在我们有了页面,需要有窗口展示出来,接下来我们需要创建一个窗口

  • 创建窗口以打开 index.html

main.js

// app 它控制应用程序的事件生命周期
// BrowserWindow 它创建和管理应用程序窗口
const { app, BrowserWindow } = require('electron');

// 添加一个createWindow()方法将index.html加载进一个新的BrowserWindow实例
const createWindow = () => {
  const win = new BrowserWindow({
    width: 800, // 窗口宽度
    height: 800, // 窗口高度
  });

  win.loadFile('index.html'); // 加载页面

  // 打开开发工具
  // win.webContents.openDevTools()
};

// 在 Electron 中，只有在 app 模块的 ready 事件被激发后才能创建浏览器窗口。
// 可以通过使用 app.whenReady() API来监听此事件。 在whenReady()成功后调用createWindow()。
app.whenReady().then(() => {
  createWindow();

  app.on('activate', () => {
    // 如果没有任何浏览器窗口是打开的，则调用 createWindow() 方法。
    if (BrowserWindow.getAllWindows().length === 0) createWindow();
  });
});

// 关闭所有窗口时退出应用
app.on('window-all-closed', () => {
  // 如果用户不是在 macOS(darwin) 上运行程序，则调用 app.quit()
  if (process.platform !== 'darwin') app.quit();
});

  • 通过预加载脚本从渲染器访问 Node.js。

预加载脚本在渲染器进程加载之前加载，并有权访问两个 渲染器全局 (例如 window 和 document) 和 Node.js 环境。

创建一个 preload.js

window.addEventListener('DOMContentLoaded', () => {
  const replaceText = (selector, text) => {
    const element = document.getElementById(selector);
    if (element) element.innerText = text;
  };

  for (const dependency of ['chrome', 'node', 'electron']) {
    replaceText(`${dependency}-version`, process.versions[dependency]);
  }
});

要想 preload.js 代码生效,我们需要将BrowserWindow构造器中将此脚本文件传入webPreferences.preload选项

// main.js

// app 它控制应用程序的事件生命周期
// BrowserWindow 它创建和管理应用程序窗口
const { app, BrowserWindow } = require('electron');
// include the Node.js 'path' module at the top of your file

const path = require('path'); //++++

// 添加一个createWindow()方法将index.html加载进一个新的BrowserWindow实例
const createWindow = () => {
  const win = new BrowserWindow({
    width: 800,
    height: 800,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'), // 预加载
    },
  });

  win.loadFile('index.html');

  // 打开开发工具
  // win.webContents.openDevTools()
};

// 在 Electron 中，只有在 app 模块的 ready 事件被激发后才能创建浏览器窗口。
// 可以通过使用 app.whenReady() API来监听此事件。 在whenReady()成功后调用createWindow()。
app.whenReady().then(() => {
  createWindow();

  app.on('activate', () => {
    // 如果没有任何浏览器窗口是打开的，则调用 createWindow() 方法。
    if (BrowserWindow.getAllWindows().length === 0) createWindow();
  });
});

// 关闭所有窗口时退出应用
app.on('window-all-closed', () => {
  // 如果用户不是在 macOS(darwin) 上运行程序，则调用 app.quit()
  if (process.platform !== 'darwin') app.quit();
});

# 打包

打包可以用electron-packager 和 electron-builder 以及electron-forge

# electron-packager

通过 JavaScript 或命令行将 Electron 应用程序打包到特定于操作系统的 bundle (. app，. exe 等)中。

更多使用请查看官方文档 electron-packageropen in new window

  • 安装

推荐安装

yarn add electron-packager --dev
1
electron-packager <sourcedir> <appname> --platform=<platform> --arch=<arch> [optional flags...]
1

sourcedir:当前目录

appname:打包后可执行文件的 app 名

--platform : 运行环境("linux" | "win32" | "darwin" | "mas" )

--archarch : 架构("ia32" | "x64" | "armv7l" | "arm64" | "mips64el" | "universal")

--electron-version: 指定 electron-version 的版本,最好是跟 package.json 中的 electron 版本保持一致

--overwrite: 打包时覆盖之前的包

--ignore: 被忽略的文件不会被进行打包

--icon: 设置应用程序的图标(linux 环境不支持此 option,需要在 BrowserWindow 中设置

...其他配置项,请查看文档

  • 使用

例如下面我想打一个 linux 环境可执行的包

// package.json
{
  ...
  "scripts": {
    "packager:linux-x64": "electron-packager . my-electron-app  --platform=linux --arch=x64 --out my-electron-app  --electron-version 25.5.0 --overwrite --ignore=/node_modules --download.mirrorOptions.mirror=https://npm.taobao.org/mirrors/electron/"
  }
  ...
}

# electron-forge

Electron Forge 是一个处理 Electron 应用程序打包与分发的一体化工具。 在工具底层，它将许多现有的 Electron 工具 (例如 electron-packager、 @electron/osx-sign、electron-winstaller 等) 组合到一起，因此不必费心处理不同系统的打包工作

有关 make 或其他 Forge API 的更多信息，请查看 Electron Forge CLIopen in new window 文档。

  • 安装
yarn add --dev @electron-forge/cli

npx electron-forge import
